# 即时配送竞品情报看板

这是一个用于跟踪闪购、即时零售、同城跑腿、即时配送平台动态的情报看板。

## 页面

- `index.html`：公开访问的网站首页。
- `auto_update_intel.py`：每日抓取候选线索，并写入页面的自动候选池。
- `.github/workflows/daily-update.yml`：每天 09:30（北京时间）自动运行抓取脚本。

## 使用方式

1. 在 GitHub 仓库中开启 Pages。
2. 选择从 `main` 分支的根目录发布。
3. 页面发布后，访问 GitHub Pages 给出的公开网址。

## 日常维护

- 页面 UI 优化：修改 `index.html`。
- 抓取范围优化：修改 `auto_update_intel.py` 里的平台词、关键词、`DATA_SOURCES` 数据源池。
- 自动更新时间调整：修改 `.github/workflows/daily-update.yml` 的 `cron` 配置。

自动抓取只进入“候选池”，仍建议由产品经理人工确认后再入库。

## 数据源池

当前脚本已覆盖 44 个固定来源，分为：

- 行业媒体：36氪、虎嗅、钛媒体、亿欧、亿邦动力、联商网、网经社。
- 科技媒体：IT之家、TechWeb、DoNews、雷峰网。
- 财经媒体：界面、第一财经、21财经、每日经济新闻、证券时报、财联社。
- 综合与地方媒体：新浪财经、中国经济网、中新网、新华网、人民网、央视网、澎湃、新京报、京报网、同花顺财经、东方财富、上观新闻、观察者网、大洋网、搜狐、网易、今日头条。
- 技术与公众号索引：淘宝技术、美团技术团队、微信公众号文章索引。
- 平台官方：美团、顺丰同城、闪送、UU跑腿、达达、京东到家、饿了么官网。

在 `DATA_SOURCES` 中新增来源时，建议补齐 `name`、`domain`、`focus` 和 `weight`。`weight` 会影响候选相关度评分，越高越容易排在前面。
